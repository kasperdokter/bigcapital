<p align="center">
  The official Bigcapital docker-compose requires building images from source. This repo provides a standalone compose that uses pre-built images only, making deployment significantly simpler — no build step needed.
</p>

> ⚠️ **This template is not officially maintained.** It may break when Bigcapital releases breaking changes. Use at your own risk.

## Deploy on Coolify

1. In Coolify, go to your project and click **New Resource**
2. Choose **Docker Compose (Empty)**
3. Paste the contents of [`bigcapital.yaml`](./bigcapital.yaml), or use the raw URL directly:
   ```
   https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_REPO/main/bigcapital.yaml
   ```
4. Fill in the required environment variables (see below)
5. Deploy

## Auto-generated by Coolify

Coolify automatically generates and manages these — you don't need to set them:

- `SERVICE_USER_MYSQL` / `SERVICE_PASSWORD_MYSQL` — database credentials
- `SERVICE_PASSWORD_MYSQLROOT` — MariaDB root password
- `SERVICE_PASSWORD_64_JWT` — JWT secret
- `SERVICE_URL_WEBAPP` — public URL of your Bigcapital instance

## Optional integrations

All disabled by default. Set only what you need:

| Integration | Variables |
|---|---|
| Exchange rates | `EXCHANGE_RATE_SERVICE`, `OPEN_EXCHANGE_RATE_APP_ID` |
| Plaid bank feeds | `BANK_FEED_ENABLED`, `PLAID_ENV`, `PLAID_CLIENT_ID`, `PLAID_SECRET`, `PLAID_LINK_WEBHOOK` |
| S3 storage | `S3_REGION`, `S3_ACCESS_KEY_ID`, `S3_SECRET_ACCESS_KEY`, `S3_ENDPOINT`, `S3_BUCKET` |
| Stripe payments | `STRIPE_PAYMENT_SECRET_KEY`, `STRIPE_PAYMENT_PUBLISHABLE_KEY`, `STRIPE_PAYMENT_CLIENT_ID`, `STRIPE_PAYMENT_WEBHOOKS_SECRET`, `STRIPE_PAYMENT_REDIRECT_URL` |
| Lemon Squeezy | `LEMONSQUEEZY_API_KEY`, `LEMONSQUEEZY_STORE_ID`, `LEMONSQUEEZY_WEBHOOK_SECRET` |
| Mail | `MAIL_HOST`, `MAIL_USERNAME`, `MAIL_PASSWORD`, `MAIL_FROM_ADDRESS`, `MAIL_PORT`, `MAIL_SECURE`, `MAIL_FROM_NAME` |

## Signup controls

By default signup is open. To restrict access:

| Variable | Default | Description |
|---|---|---|
| `SIGNUP_DISABLED` | `false` | Set to `true` to disable all signups |
| `SIGNUP_ALLOWED_DOMAINS` | — | Comma-separated list of allowed email domains |
| `SIGNUP_ALLOWED_EMAILS` | — | Comma-separated list of allowed email addresses |
| `SIGNUP_EMAIL_CONFIRMATION` | `false` | Require email confirmation on signup |
